# Extra Saucy Swift Audio APIs

Hello There! I'm [Carlos](https://carlosmbe.com), and this is the repo for [my talk](https://www.youtube.com/watch?v=HzbDJL0_yLY&t=18957s) at **[Deep Dish Swift 2026](https://deepdishswift.com)** (April 12-14, Chicago).

The talk covers three layers of Apple's on-device speech intelligence stack and how they fit together:

1. **[SpeechAnalyzer](https://developer.apple.com/documentation/speech/speechanalyzer)** (`import Speech`) — Apple's new [WWDC25](https://developer.apple.com/videos/play/wwdc2025/277/) transcription framework
2. **[FluidAudio](https://github.com/FluidInference/FluidAudio)** — Community-built speaker diarization, ANE optimized
3. **[Foundation Models](https://developer.apple.com/documentation/FoundationModels)** (`import FoundationModels`) — On device LLM structured output

All processing runs entirely on-device. No cloud, no API keys.
